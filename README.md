multi-terminal workspaces with shared context, agent orchestration, and a persistent brain.

## Stack

- **Frontend:** React 18 + Vite + TypeScript + Tailwind v4 ‚Äî port 5180
- **Server:** Express + ws (WebSocket) ‚Äî port 4000, proxied via `/api` and `/ws`
- **Agent runtime:** Claude Code CLI / Agent SDK (subscription auth ‚Äî no API keys)

## Run

```
npm install
npm run dev        # server :4000 + frontend :5180
```

Or double-click `Start 9 - Christopher OS.bat` in the parent folder.

Notes:

- `npm install` from the repo root is **required** before the first run ‚Äî it installs both
  workspaces (`frontend/`, `server/`). The server cannot start without it (`express`, `ws`,
  `chokidar`, `node-pty` are all runtime imports).
- `node-pty` is pinned to the exact prerelease `1.2.0-beta.12` (caret ranges behave
  surprisingly across prerelease tags) and needs a native build toolchain on Windows
  (prebuilt binaries usually cover current Node; otherwise install VS Build Tools).
- The app is currently **dev-proxy-only**: the Vite dev server proxies `/api` and `/ws`
  (which prefix-matches `/ws/terminal`) to the Express server on :4000. The Express app
  serves no static files ‚Äî for a production build you'd add `express.static(frontend/dist)`
  + an index fallback while keeping the `/ws` upgrade routing.
